	Filter can be used to display Requirements by certain criteria.
	<ul class="arrow-list">
		<li><strong>Filter by cross-cutting theme</strong><br />For retrieval
			of cross-cutting variables, e.g. in the cryosphere, the concept of
			'themes' is used. A variable can be attached to none, one or more
			themes. When selected, all requirements defined for variables in the
			respective theme are returned.</li>
		<li><strong>Filter by Application area</strong><br />Select one ore
			more application areas for which the requirements are defined.</li>
		<li><strong>Filter by vertical layer(s)</strong><br />A layer refers
			to the vertical domain in which the requirement is defined. Hover of
			the acronym to see the definition.</li>
		<li><strong>Filter by horizontal coverage(s)</strong><br />Horizontal
			coverage refers to the horizontal domain for which the requirement is
			needed, i.e. globally, regionally, for specific points, or over land
			or over the ocean.</li>
	</ul>
	Generally, if no item is selected, all are returned. <br />The <strong>export
		functionality </strong>takes the current filter into account
